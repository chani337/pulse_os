# Pulse OS, 아직 개발 중인 단계입니다.

개인 생활 리듬 기반 작업 관리 시스템

## 프로젝트 개요

Pulse OS는 개인의 생활 리듬에 기반한 작업 관리 시스템입니다.

해야 할 모든 일을 관리하는 것이 아니라, **지금 당장 할 수 있는 일**에 집중합니다.

시스템은 완전히 오프라인으로 동작하며, 프라이버시를 최우선으로 합니다.

## 배경

대부분의 기존 할 일 앱들은 사용자에게 긴 작업 목록을 제공하여 과부하를 줍니다.

이는 종종 심리적 피로와 앱 포기로 이어집니다.

Pulse OS는 리듬, 에너지, 컨텍스트를 중심으로 생산성을 재설계합니다.

## 목표

- 컨텍스트 인식 생산성 시스템 구축
- 실행률 향상
- 미루기를 분석 가능한 데이터로 전환
- 일일 작업과 연간 인생 목표 연결
- 로컬 우선 아키텍처 유지

## 핵심 개념

Pulse OS는 작업을 관리하지 않습니다.

**일일 리듬을 관리합니다.**

핵심 질문은 다음과 같습니다:

> "지금 당장 현실적으로 할 수 있는 일은 무엇인가?"

## 주요 기능

- **개인화된 온보딩 시스템**: 사용자의 생활 패턴과 목표를 수집
- **컨텍스트 기반 작업 필터링**: 현재 상황에 맞는 작업만 표시
- **일일 포커스 시스템**: 하루에 집중할 상위 3개 작업
- **미루기 사유 추적**: 작업을 미룬 이유를 기록하여 패턴 분석
- **템플릿 기반 작업 분해**: 큰 목표를 실행 가능한 단계로 분해
- **자동화된 리뷰 인사이트**: 주기적인 성과 분석 및 개선 제안

## 온보딩 시스템

온보딩 플로우에서 다음 정보를 수집합니다:

- 생활 유형
- 주요 활동 및 시간 블록
- 연간 목표
- 에너지 패턴
- 사용 선호도

모든 정보는 기기에만 저장됩니다.

## 추천 로직

작업은 다음 요소들을 기반으로 추천됩니다:

- 사용 가능한 시간
- 에너지 수준
- 예상 소요 시간
- 컨텍스트 제약 조건

## 기술 스택

- **Flutter**: 크로스 플랫폼 모바일 프레임워크
- **Hive**: 로컬 데이터베이스
- **Clean Architecture**: 계층화된 아키텍처
- **Offline First**: 오프라인 우선 설계
- **Optional AI Extension**: 선택적 AI 확장 모듈

## 프로젝트 구조

```
lib/
├── app/              # 애플리케이션 설정 및 라우팅
├── core/             # 공유 유틸리티 및 상수
├── domain/           # 비즈니스 로직 (모델, 리포지토리, 유스케이스)
├── data/             # 로컬 스토리지 구현
├── engines/          # 규칙 및 템플릿 로직
├── features/         # UI 모듈 (온보딩, 홈, 인박스, 플랜, 리뷰, 설정)
└── ai/               # 선택적 LLM 모듈
```

## 시작하기

### 필수 요구사항

- Flutter SDK (>=3.0.0)
- Dart SDK
- Android Studio / Xcode (모바일 개발용)
- VS Code 또는 Android Studio (에디터)

### Flutter 설치 확인

터미널에서 Flutter가 설치되어 있는지 확인:
```bash
flutter --version
```

Flutter가 설치되어 있지 않다면:
- macOS: `brew install --cask flutter`
- 또는 [Flutter 공식 사이트](https://flutter.dev/docs/get-started/install)에서 설치

### 설치 및 실행

1. **의존성 설치**:
```bash
flutter pub get
```

2. **사용 가능한 디바이스 확인**:
```bash
flutter devices
```

3. **앱 실행**:
```bash
# iOS 시뮬레이터 (macOS만)
flutter run -d ios

# Android 에뮬레이터
flutter run -d android

# Chrome (웹)
flutter run -d chrome

# 기본 디바이스로 실행
flutter run
```

### 개발 모드 실행

핫 리로드가 가능한 개발 모드로 실행:
```bash
flutter run
```

실행 중:
- `r`: 핫 리로드
- `R`: 핫 리스타트
- `q`: 종료

### 온보딩 화면 확인

앱이 실행되면 기본적으로 홈 화면이 표시됩니다. 온보딩 화면을 보려면:

1. 앱 내에서 `/onboarding` 라우트로 이동하거나
2. `lib/app/routes.dart`에서 초기 라우트를 변경:
```dart
initialRoute: '/onboarding',  // '/' 대신
```

### 문제 해결

**의존성 오류가 발생하면**:
```bash
flutter clean
flutter pub get
```

**빌드 오류가 발생하면**:
```bash
flutter doctor
```
명령어로 환경 설정을 확인하고 문제를 해결하세요.
